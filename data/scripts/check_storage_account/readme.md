帮我实现以下CSPM需求：
【规则名称】：存储帐户允许使用永久访问密钥，存在凭证泄露风险
【风险等级】：Base: 7.0, Net: 2.0, IAM: 1.5, Res: 1.5, Score: 31.5, Level: High
【规则描述】：Azure 存储帐户是微软云提供的可大规模缩放的对象存储服务，常用于存放备份数据、静态网站内容和日志文件。此规则检查存储帐户是否启用了“共享密钥授权”。共享密钥是永久有效的静态凭证，一旦泄露，相当于交出了整个存储帐户的最高管理权限。在现代云应用中，推荐使用有时效性、可精细化授权的 Azure AD 凭证和共享访问签名 (SAS) 进行访问，以遵循最小权限原则。
【潜在后果】：
攻击者一旦获取共享密钥，可以不受限制地窃取、篡改或删除存储帐户中的所有数据，例如客户资料、财务报表或数据库备份，可能导致核心业务中断和重大数据泄露事故。黑客通常通过扫描公开的代码仓库、反编译移动应用或利用服务器漏洞来寻找硬编码在配置文件中的共享密钥，获得后即可利用自动化工具进行“脱库”或植入勒索软件。

【修复建议】：
1. 登录到 Azure 门户，导航到“存储帐户”服务。
2. 点击目标存储帐户 {instance_id}。
3. 在左侧菜单的“设置”部分，选择“配置”。
4. 在主窗格中，找到“允许存储帐户密钥访问”选项，并将其设置为“已禁用”。
5. 单击“保存”。
您可以在以下链接直接访问配置页面：https://portal.azure.com/#resource/subscriptions/{subscription_id}/resourceGroups/{resource_group_name}/providers/Microsoft.Storage/storageAccounts/{instance_id}/configuration

注意事项：禁用共享密钥授权前，请务必全面排查并修改所有依赖此密钥进行访问的应用程序或服务。建议将认证方式迁移至基于 Azure Active Directory 的 RBAC 或使用有时效性的共享访问签名 (SAS)，以避免业务中断。
